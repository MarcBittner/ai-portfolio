Help · burnrate

What it is

An instrumented Flask SRE service: real RED metrics (Prometheus exposition), availability & latency SLIs/SLOs with an error budget, a canonical multiwindow burn-rate alerting policy, and an LLM-backed incident summary. A deterministic fault is injectable so you can burn and recover the budget.

How to use

  1. Drive load — click Drive load to send synthetic traffic and populate the SLO windows.
  2. Inject an incident — Inject fast burn (errors + latency → page) or slow burn (errors → ticket); watch the budget bar burn down.
  3. Generate a summary — click Generate incident summary (LLM) for a severity, action, and next steps.
  4. Recover / reset — clears the fault and starts a fresh window.
