Help · llm-gateway

What it is

A provider-agnostic LLM gateway that puts governance on the request path: every completion runs firewall → redact → route → output firewall → redact → tamper-evident audit. The deterministic guardrails need no model; routing falls back to an offline mock when no provider is reachable.

How to use

  1. Send a prompt — type or pick an example (benign · PII · injection) in Policy, then click Send through gateway.
  2. Watch the path — the Request path panel shows each stage: input scan, redact, route, output scan, result.
  3. Pick routing — open ⚙ Configuration to set the routing mode and an optional model override.
  4. Verify the audit — click Verify chain; try Tamper to break it, then re-verify.
