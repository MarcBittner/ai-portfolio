Help · rtc-guard

What it is

A real-time access guard: it mints scoped, signed WebRTC room tokens (least-privilege grants with a TTL), runs an adversarial breaker suite that proves common attacks are rejected, and ships an LLM grant auditor that flags over-permissioning in plain English. The security core (mint/verify) is deterministic; the model only explains and judges.

How to use

  1. Mint a token — set identity, room, grant template, and TTL in Mint a scoped token, then Mint token to see the signed grant.
  2. Verify a token — paste a token (or use the minted one) and an expected room in Verify a token to check signature, TTL, and room scope.
  3. Run the adversary — the Adversarial suite panel shows each attack and why it was blocked.
  4. Lint a grant — toggle capabilities in Least-privilege grant auditor and Audit grant for findings + severities.
