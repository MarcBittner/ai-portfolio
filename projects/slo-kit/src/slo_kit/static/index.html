Help · slo-kit

What it is

An instrumented reference SRE service: RED metrics (rate/errors/duration), availability & latency SLOs with error budgets and burn rate, OpenTelemetry-style traces, and an on-demand incident you can inject and recover. The model only writes the on-call narrative; severity is classified deterministically from the SLO math.

How to use

  1. Run load — click Run load in Operator controls to drive healthy traffic and populate the SLOs.
  2. Inject an incident — set error % and added latency, click Inject incident + load to burn the error budget.
  3. Generate a summary — click Generate incident summary to compress the live state into a severity, narrative, and runbook steps.
  4. Reset — click Reset to clear the fault and recover the budget.
