What it is

A high-volume political-contributions data service: a synthetic dataset in SQLite partitioned by election cycle, a before/after query-plan artifact showing the covering-index win, FEC-style committee rollups, and natural-language → guarded-SQL questions.

How to use

  1. Read the dataset — the Dataset cards and Query plan panel show row counts and the full-scan → index-search diff.
  2. Run a rollup — pick a cycle in FEC-style rollup and click Aggregate to see per-committee totals and elapsed time.
  3. Run the surge — set a query count in End-of-quarter surge and click Run surge for throughput and tail latency.
  4. Ask in plain English — type a question in Ask in plain English; a model writes SQL, a guard runs it only if it is a single read-only SELECT.
